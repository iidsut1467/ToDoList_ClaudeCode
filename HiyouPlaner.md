# HiyouPlaner（費用プランナー）開発メモ

このファイルは **別のPC（Mac / Windows）でも開発を引き継げるように**、これまでの実装内容と
これからの予定をまとめた記録。環境構築・起動・GitHub への反映手順は [CLAUDE.md](CLAUDE.md) を参照。

別環境では先に `git clone https://github.com/iidsut1467/ToDoList_ClaudeCode.git` してから作業する。

## アプリ概要
学習用の ToDo アプリを拡張中。タスクに期限・優先度などの属性を持たせ、
**木構造（根・節・葉）** で細分化できる。今後は「費用プランナー」として、
**費用・日時・ジャンル（分類）** も扱える方向へ拡張していく。

## 技術構成（要点）
- フロント: `index.html` / `app.js` / `style.css`。`app.js` が `api.php` に fetch で通信。
- バックエンドは同じ仕様を2種類: `api.php`（PHP）/ `server.rb`（Ruby）。**必ず両方を同仕様に揃える**。
- データは `data.json` に保存（自動生成・`.gitignore` で除外）。
- データ1件の形:
  `{ id, text, done, dueAt, priority, quick, difficulty, estimateMin, parentId }`
- 起動: `php -S localhost:8000` → ブラウザで http://localhost:8000/ （停止は Ctrl+C）

## これまでに実装した機能
- **属性**: 期限 `dueAt` / 優先度 `priority`(high,mid,low) / 難度 `difficulty`(easy,normal,hard) /
  所要時間 `estimateMin`(分) / 前提「すぐ終わる?」`quick`(真偽)。
  - 「すぐ終わる=はい」のときは難度・所要時間の入力欄を隠す。
- **自動並べ替え**: 期限が近い順 →（同じなら）優先度順。完了は下へ。
- **残り日数表示**: 「あと◯日／今日まで／◯日超過」。近い=橙・超過=赤で色分け。
- **表（テーブル）形式**で一覧表示。横幅はウィンドウいっぱい。
- **木構造（根・節・葉）**: `parentId` で親子関係。各行の「＋子」ボタンで子タスク追加。
  子は字下げ＋「└」で階層表示。並べ替えは同じ階層の中で効く。
- **名前の編集**: 各行「編集」ボタン →（PUT で text のみ更新）。
- **削除**: 親を消すと子孫もまとめて削除＋確認ダイアログ。

## これからの予定（設計中）
方向性: タスクに **費用・日時・ジャンル（分類）** を加える。さらに、
**未入力の項目は表示せず（無駄な縦長・空欄を作らない）、入力したものだけをスクロール先で見られる** UI にする。

用語の整理（重要・ユーザー確定）:
- このアプリは「やること」管理というより **お金（入金・支払い）の管理＝費用プランナー** に寄せていく。
- 既存の **「期限」`dueAt` は「入金の期限」** を意味する（いつまでに入金/支払うか）。
- **「費用」は具体的にかかる金額**（お金）。

追加したい項目:
- **費用（金額）**: 具体的にかかる金額。整数の円を想定（要確認: 桁区切り表示など）。
- **日時**: **イベントや具体的な日付のある予定**（例: 発売日・開催日・約束）。
  入金期限 `dueAt` とは**別物**として持つ。時刻まで持つかは要確認。
- **ジャンル**: **趣味 / 仕事** などの分類。さらに **趣味の中に「ゲーム」「漫画」**、
  その中の **具体的な物（タイトル等）** まで、**階層的に細かく分類** したい意向。
  → タスクの木構造（根・節・葉）と同じ考え方で、ジャンルも何段でも掘り下げられる形が望ましい。
  選択式か自由入力か、候補リストは未確定。

表示の方針（ユーザー要望）:
- 入力されるまでは表示を出さない（空の表・空の項目で場所を取らない）。
- 入力された詳細はスクロールした先で見られるようにする。

既存ロードマップの積み残し:
- リマインダー／タイマー（ブラウザ通知）。
- カレンダー連携（外部API・認証が重いので最後）。

## 設計上の共通ルール
- 新しい項目は「**無ければ既定値**」で扱い、古いデータでも壊れないようにする（後方互換）。
- `api.php` と `server.rb` は常に同じ仕様に保つ。不正値・未指定はサーバー側で既定値に補正。
- フロントは `app.js` の表示（`render`/`buildRow`）と入力フォーム、サーバー側の保存項目を**セットで**増やす。

## 次回ここから（未確定の確認事項）
1. **費用（金額）** の入力形式（整数の円でよいか、桁区切り表示は要るか）。
2. **ジャンル** は選択式か自由入力か。分類の候補は何にするか。
3. **日時** は入金期限に時刻を足す意味か、別の項目か。
4. 「未入力は隠す／入力したらスクロールで見える」の具体的な見せ方（行を折りたたむ／詳細を別領域に出す 等）。

確定済み:
- 「期限」=入金の期限。「費用」=具体的にかかる金額。
